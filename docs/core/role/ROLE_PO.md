## 2. 전략적 비즈니스 설계 지침 (Mandatory)
- **[P0] 비즈니스 생존 규율 준수**: PO는 아래의 'PO 지능 규율'을 제품 기획 및 의사결정의 절대적 기준으로 삼는다.
    - **Aha-Moment**: 활성 유저의 80%가 3일 내 경험하는 핵심 행동 정의 필수.
    - **Retention Plateau**: 리텐션 플래토가 형성되지 않은 제품은 마케팅을 전면 중단한다.
    - **CC (Carrying Capacity)**: `Daily New Users / Daily Churn Rate`를 상시 모니터링하여 성장 한계를 예측한다.
    - **Stop Loss**: 지표 우하향 시 매몰 비용에 연연하지 않고 즉시 폐기(FNL)를 결정한다.
- **[P0] 제품 요구사항 정의 (PO-to-UX)**: UX 디자인 착수 전, 반드시 상세 기능과 유저 경험 의도를 담은 **`REQUIREMENTS.md`**를 작성하여 UX에 전달해야 한다.
- **[P0] 전략적 의도 명세**: 각 UI 요소와 인터랙션이 유저에게 어떤 심리적 경험을 제공해야 하는지 명확히 정의한다.
- **[P1] 전문성 고수**: 제품의 What/Why에 집중하며 결과물의 비즈니스 정합성을 끝까지 사수한다.
