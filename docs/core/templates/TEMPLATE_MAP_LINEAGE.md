# 🗺️ 가재 컴퍼니 유니버설 템플릿 계보도 (Universal Template Lineage Map)

본 문서는 가재 컴퍼니의 모든 문서가 어떤 공정에서 생성되고, 어떤 데이터를 참조하며, 최종적으로 어떤 산출물을 만드는지 그 '지능의 흐름'을 정의합니다.

## 1. 지능 데이터 파이프라인 (Intelligence Pipeline)

| 문서명 (Template) | 공정 단계 (Process) | 주관 가재 | 핵심 목적 (Why) | 후속 산출물 (Next Step) |
| :--- | :--- | :--- | :--- | :--- |
| **PF 기획서** | 1. PF | PO | 비즈니스 가치 검증 및 기능 정의 | 1-Pager 기획안 |
| **1-Pager 기획안** | 2. FBS | PO | 제품 비전 및 성공 지표 구체화 | 제품 요구사항 정의서 |
| **제품 요구사항** | 2. FBS | PO | 상세 기능 및 유저 경험 의도 정의 | 디자인 시스템, UI 명세 |
| **디자인 시스템** | 3. RFD | UX | 프로젝트 고유의 미학적 규격 수립 | UI 명세, 개발 이관 가이드 |
| **UI/UX 명세서** | 4. FBD | UX | 1px 단위의 화면 설계 및 플로우 | 개발 이관 가이드 |
| **개발 이관 가이드** | 4. FBD | UX | 기술적 구현 데이터(Tailwind, Z) | 소스 코드 (FUE) |
| **인시던트 보고서** | Incident | 수행원 | 장애 원인 분석 및 재발 방지 | 추가 정정 미팅록 |
| **공정 승인 관문** | 전 단계 | PM | CEO 최종 승인 및 단계 전환 관리 | 다음 공정 태스크 |

## 2. 참조 및 종속성 규칙 (Lineage Rules)
1. **Inheritance (상속)**: 모든 하위 문서는 반드시 직전 단계 문서의 **[Full URL]**과 **[Commit Hash]**를 상단에 참조 명시해야 한다.
2. **Copy-Only**: 모든 신규 문서는 `docs/core/templates/`에 박제된 원본 템플릿을 복제하여 생성하며, 포맷 변경을 절대 금지한다.
3. **Traceability (추적성)**: 특정 문서의 결정이 변경될 경우, 계보를 따라 연결된 모든 하위 문서의 태스크보드에 `[영향분석 대응]` 태스크가 자동 생성되어야 한다.
