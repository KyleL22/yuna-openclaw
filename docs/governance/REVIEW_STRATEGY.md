## 1. 가짜 리뷰 전수 조사 전략
- **대상:** 서비스 소스 코드 및 `docs/` 내 기획 문서
- **결과:** 현재까지 코드 및 문서 내에 하드코딩된 '가짜 리뷰' 또는 '가짜 사용자 후기' 요소는 발견되지 않음. 
- **조치:** 향후 개발 및 기획 단계에서 AI 생성형 가짜 리뷰 도입을 엄격히 금지함 (LEGAL/PO 상시 감시).

## 2. '리얼 리뷰' 데이터 수집 계획 (Real Evidence Strategy)
단순한 텍스트 후기가 아닌, 서비스의 신뢰도를 증명할 수 있는 **'리얼 증명 데이터'**를 수집하여 브랜딩에 활용함.

### 2.1. 수집 데이터 세트 (The Trinity Set)
1. **[Past]** 유저 원천 데이터 (유저 제공)
2. **[Now]** 서비스 AI 결과물 (서비스 생성)
3. **[Future]** 실제 매칭 결과물 (협력 유저 제공)

### 2.2. 추진 단계
- **Phase 1 (지인/앰버서더):** 대표님 및 가재 부대원의 실제 지인들을 대상으로 베타 테스트 진행 및 '리얼 데이터 세트' 수집 (동의 필수).
- **Phase 2 (랜딩 페이지 반영):** 수집된 리얼 매칭 데이터를 랜딩 페이지 하단에 '리얼 증명' 섹션으로 배치하여 신뢰도 극대화.
- **Phase 3 (커뮤니티 확산):** 실제 만족한 유저들이 자신의 SNS에 [Past vs Now vs Future] 데이터를 비교 포스팅하도록 유도하는 챌린지 전개.

## 3. LEGAL/PO 검토 사항
- **초상권 및 개인정보:** 수집된 데이터는 반드시 마케팅 활용 동의서를 징구함.
- **진실성 보장:** 모든 마케팅용 리뷰는 실제 서비스 이용 결과임을 LEGAL이 보증함.
