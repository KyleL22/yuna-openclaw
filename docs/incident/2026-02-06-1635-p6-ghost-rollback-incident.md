## 3. 근본 원인 (Root Cause)
1. **분산된 수정 세션**: 서브에이전트 구현 세션과 중앙 OS 세션이 동시에 `README.md`와 `docs/`를 건드리며 발생한 '경합 조건(Race Condition)'.
2. **지능의 확증 편향**: 수행원이 최신 헌법 v26.0을 수립하며 `BYLAWS.md` 등 하위 법령을 당연한 것으로 간주하고, 인덱싱에서 누락시킨 '연산 소홀'.

## 4. 유저 영향 (User Impact)

## 5. 5-Whys 분석
1. **Q**: 왜 README가 롤백됐나? -> **A**: 서브에이전트의 커밋을 병합하는 과정에서 수행원의 최신 수정본이 덮어씌워졌기 때문.
2. **Q**: 왜 덮어씌워졌나? -> **A**: `git push --force`에 준하는 `rsync` 기반 덮어쓰기 로직이 세션 간의 동기화를 파괴했기 때문.
3. **Q**: 왜 하위 법령이 안 보이나? -> **A**: 헌법의 고도화에만 집중하여 하위 법령 레이어를 참조 목록에서 누락시키는 '지능의 협착' 발생.
4. **Q**: 왜 시스템이 롤백을 감지 못했나? -> **A**: 파일의 '내용 밀도' 변화를 실시간으로 모니터링하는 가디언이 README에는 적용되지 않았기 때문.
5. **Q**: 왜 적용 안 했나? -> **A**: 리드미를 '관리 문서'가 아닌 '홍보 문서'로 과소평가하여 무결성 감시망에서 제외했기 때문.
